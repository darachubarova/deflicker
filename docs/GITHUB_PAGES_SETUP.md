# Настройка GitHub Pages для Mask Stabilization

## Инструкция по включению GitHub Pages

После того как Pull Request будет смержен в ветку `main`, необходимо настроить GitHub Pages для публикации сайта-презентации.

### Шаг 1: Перейдите в настройки репозитория

1. Откройте репозиторий на GitHub: https://github.com/darachubarova/defliker
2. Нажмите на вкладку **Settings** (Настройки)

### Шаг 2: Найдите раздел Pages

1. В левом меню найдите раздел **Pages** (обычно в категории "Code and automation")
2. Кликните на **Pages**

### Шаг 3: Настройте источник публикации

В разделе **Build and deployment**:

1. **Source**: Выберите **Deploy from a branch**
2. **Branch**: 
   - Выберите **main** (основная ветка)
   - Выберите папку **/docs** (где находится сайт)
3. Нажмите **Save**

### Шаг 4: Дождитесь публикации

1. GitHub автоматически запустит процесс публикации
2. Через 1-2 минуты ваш сайт будет доступен по адресу:
   ```
   https://github.com/darachubarova/defliker/
   ```
3. Вы увидите зелёное уведомление с ссылкой на опубликованный сайт

### Шаг 5: Проверьте работу сайта

1. Перейдите по ссылке https://github.com/darachubarova/defliker/
2. Убедитесь, что все секции отображаются корректно
3. Проверьте навигацию между секциями
4. Проверьте адаптивность на мобильных устройствах

## Дополнительная информация

### Автоматическое обновление

После настройки GitHub Pages будет автоматически обновляться при каждом push в ветку `main`:
- Любые изменения в папке `/docs` будут автоматически опубликованы
- Обновление занимает 1-2 минуты

### Пользовательский домен (опционально)

Если у вас есть собственный домен:
1. В разделе **Pages** найдите **Custom domain**
2. Введите ваш домен (например, `mask-stabilization.example.com`)
3. Настройте DNS записи согласно инструкциям GitHub

### Устранение проблем

**Сайт не отображается:**
- Проверьте, что файл `/docs/index.html` существует в ветке `main`
- Убедитесь, что выбрана правильная ветка и папка в настройках
- Подождите несколько минут для первой публикации

**404 ошибка:**
- Убедитесь, что путь к файлу правильный
- Проверьте, что папка называется именно `docs` (строчными буквами)

**Стили не загружаются:**
- Все стили встроены в HTML, проблем быть не должно
- Проверьте консоль браузера на наличие ошибок
